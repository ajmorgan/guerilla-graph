## Feature Summary
Complete implementation of --description-file flag for reading markdown files into plan/task descriptions.

## Goals
- Fix critical memory leak in existing task new implementation
- Add memory ownership tracking to distinguish borrowed vs owned descriptions
- Implement --description-file for task update, plan new, and plan update
- Add mutual exclusivity validation to prevent conflicting flags
- Add comprehensive tests for file reading functionality
- Update documentation to reflect complete feature support

## Scope
**In Scope:**
- Memory leak fix (existing task new)
- Ownership tracking (description_owned flag)
- Mutual exclusivity validation (--description + --description-file)
- Feature implementation for 4 commands (task new/update, plan new/update)
- Unit and integration tests
- Documentation updates

**Out of Scope (Non-Goals):**
- Extracting utility function (inline reading is simple and clear)
- Changing 10MB size limit (current limit is appropriate)
- Adding stdin support (--description-file -)
- Editor integration (--edit flag)
- YAML frontmatter validation (deferred to Phase 2 per CLAUDE.md)

## Architecture Overview
**Data Flow:**
1. CLI parsing → Parse --description-file flag, extract file path
2. File reading → Call std.fs.cwd().readFileAlloc() with size limit
3. Ownership tracking → Set description_owned = true in parsed args
4. Command execution → Pass description to storage layer
5. Cleanup → Free owned description in defer block

**Key Design Decisions:**
- Inline file reading vs utility function: Simplicity over premature abstraction
- Strict mutual exclusivity: Clear error message prevents user confusion
- Keep 10MB size limit: Appropriate for typical use cases (<100KB)
